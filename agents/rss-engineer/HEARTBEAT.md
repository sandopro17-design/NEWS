# HEARTBEAT — RSS Engineer

Checklist a ogni wake:

1. Lavora prima su task `in_progress`.
2. Verifica stato ingestion/job aperti e possibili anomalie.
3. Applica fix mirati su dedupe, parsing, mapping taxonomy.
4. Valida impatto con test/controlli dati disponibili.
5. Aggiorna issue con:
   - modifica eseguita
   - evidenza tecnica
   - rischio residuo
6. Se bloccato, marca `blocked` con owner+ETA.

## Regole anti-spreco

- Niente tuning prematuro fuori incidenti/requisiti attivi.
- Evita commenti ripetuti senza nuovi segnali.

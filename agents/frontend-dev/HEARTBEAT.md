# HEARTBEAT — Frontend Dev

Checklist a ogni wake:

1. Prendi task `in_progress` prima dei `todo`.
2. Leggi acceptance criteria e vincoli UX prima di toccare codice.
3. Implementa cambi minimi e verificabili.
4. Esegui validazione locale per quanto applicabile:
   - lint
   - test mirati
   - build rapida
5. Aggiorna issue con:
   - cosa e stato fatto
   - evidenza di verifica
   - prossimi passi
6. Se bloccato, imposta `blocked` con owner+ETA.

## Regole anti-spreco

- Evita refactor ampi non richiesti dal task.
- Non ripetere update uguali senza novita reali.
